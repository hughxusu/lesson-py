# 类型注解

在Python中变量可以任意改变类型：

* 这样处理的好处是，编写程序自由度比较高。
* 坏处是容易产生混淆，产生类型错误，对于大型项目的维护不友好。

```python
num = 100
print(f'num 的值为: {num}')
num = 'hello'
print(f'num 的值为: {num}')
```

为了解决上面的问题Python引入了类型注解。

类型注解（Type Hinting）是一种为函数参数、返回值或变量添加数据描述信息（数据类型）的语法。

* 3.9版本，Python注解基本完善。
* 3.14版本后，注解已经达到完全体，是非常稳定且标准的形式了。

```python
num: int = 100
print(f'num 的值为: {num}')
```

* `num: int`类型注解，表示`num`的数据类型为整数。

类型注解虽然可以表示数据类型，但不会像C/C++或Java编程语言一样，使用错误的类型会导致程序无法执行。

```python
num: int = 'hello'
print(f'num 的值为: {num}')
```

> [!think]
>
> Python作为解释性语言，最大的特点就是不需要指定变量的类型，为什么还需要类型注解呢？

1. 对于大型项目，类型注解可以尽量避免开发中的类型错误。
1. 类型注解传递了一种编程理念“代码即文档”。
1. 借助类型注解可以使用第三方库，完成数据校验等功能，如：`Pydantic`库。
1. 类型注解可以在编程时，出现代码提示。

## 类型注解的使用

代码中所有用到的类都可以作为类型注解，包括：

1. Python的内置类型。
2. 标准库中所有了类。
3. 第三方开源库中使用的类。
4. 自定义的类。

基本数据类型

```python
num: int = 100
print(f'num 的值为: {num}')
message: str = '此情可待成追忆，只是当时已惘然。'
print(f'message 的值为: {message}')
is_active: bool = True
print(f'is_active 的值为: {is_active}')
```

列表与元组类型

```python
sites: list = ['Google', 'Runoob', 'Wiki']
print(sites)
colors: list[str] = ['red', 'green', 'blue', 'yellow', 'white', 'black']
print(colors)
webs: tuple = 'Facebook', 'Twitter'
print(webs)
nums: tuple[int, int, int] = (100, 200, 300)
print(nums)
```

* `sites: list`定义容器数据类型的注解。
* `colors: list[str]`定义容器数据类型时，可以指定容器中数据的类型。
* `tuple`容器数据类型指定元组内类型时，需要指定每一个值的类型`nums: tuple[int, int, int]`

字典数据类型

```python
person: dict[str, str | int] = {
  'name': 'Bob',
  'age': 25,
  'job': 'Dev',
  'city': 'New York',
  'email': 'bob@web.com'
}
print(person)
```

* `dict[str, str | int]`字典数据类型设置时，可以同时设置键和值的类型。
* `str | int`表示多重数据类型，多个数据类型可以使用`|`（或）联系在一起。

使用`typing.Literal`可以为变量指定某些具体值为类型，`Literal`表示字面量类型。

```python
from typing import Literal

gender: Literal['male', 'female'] = 'male'
print(gender)

gender = 'kid'
print(gender)
```

* `Literal['male', 'female']`表示`gender`的值只能是`'male'`和`'female'`，作用类似于枚举。

标准库中的数据类型

```python
from datetime import datetime
from enum import Enum

now: datetime = datetime.now()
print(now)

class Status(Enum):
    OPEN = 1
    CLOSED = 2

status: Status = Status.OPEN
print(status)
```

自定义的类

```python
from __future__ import annotations

class TodoItem:
    def __init__(self, desc: str):
        self.desc = desc
        self.is_completed: bool = False

    def __str__(self):
        return f'{"✅" if self.is_completed else "⭕️"}-{self.desc}'

    @classmethod
    def from_dict(cls, item_dict: dict[str, str | bool]) -> TodoItem:
        item = cls(str(item_dict['desc']))
        item.is_completed = bool(item_dict['is_completed'])
        return item

    
class TodoList:
    def __init__(self):
        self.__items: list[TodoItem] = []
```

* `from __future__ import annotations`可以保证在`TodoItem`没有定义完成时，在`from_dict`方法中提前使用`TodoItem`。
* `-> TodoItem`表示返回值数据类型。
* `self.__items: list[TodoItem]`在其他类中可以使用`TodoItem`类型。

使用`type`可以用来创建类型别名，需要Python版本3.12+

```python
type Numeric = int | float

index: Numeric = 6
print(index)

pi: Numeric = 3.14
print(pi)
```

使用`typing.Callable`函数类型

```python
from math import sin
from typing import Callable

type Number = int | float

def add(x: Number, y: Number, f: Callable[[Number], float]) -> float:
    return f(x) + f(y)

print(add(-5, 6, abs))
print(add(-5, 6, sin))
```

* `Callable[[Number], float]`中`[[Number], float]`用于设置函数参数和返回值的类型
  * `[Number]`表示所以参数的数据类型。
  * `float`表示返回值的数据类型

泛型用于保证不同变量属于同一种类型

```python
def add(x: str|int, y: str|int) -> str|int:
    return x + y

print(add('hello', 'world'))
print(add(1, 2))
print(add(1, 'hello'))
```

* 上面的代码`add(1, 'hello')`时，会报错。

如果希望兼容`str`和`int`数据类型，且输入输出保持一致，可以使用泛型。泛型可以使用`typing.TypeVar`定义

```python
from typing import TypeVar

T = TypeVar('T', str, int)

def add(x: T, y: T) -> T:
    return x + y

print(add('hello', 'world'))
print(add(1, 2))
```

* `TypeVar('T', str, int)`指定泛型，且泛型为`str`和`int`
* 上面的写法可以同一定义泛型。

在Python 3.12后可以写为

```python
def add[T: (str, int)](x: T, y: T) -> T:
    return x + y

print(add('hello', 'world'))
print(add(1, 2))
```

可以使用`typing.Any`定义任意数据类型，广泛使用`Any`类型就失去了类型注解的意义

```python
from typing import Any

person: dict[str, Any] = {
  'name': 'Bob',
  'age': 25,
  'job': 'Dev',
  'city': 'New York',
  'email': 'bob@web.com'
}
print(person)
```

类型注解的范围
$$
\text{具体类型}(\text{int/str})<
\text{联合类型}(\text{int|str})<
\text{泛型}(\text{int|str})<
\text{object}<
\text{Any}
$$

> [!note]
>
> 在类型注解的过程中，要尽可能的追求更小的范围，范围越小，代码越健壮。

# 类型注解原则

在类型注解使用中输入应尽可能宽松，输出应尽可能明确。

```python
def avg(items: list[int]) -> float:
    total = 0
    count = 0
    for item in items:
        total += item
        count += 1
    return total / count

reuslt = avg((1, 2, 3))
print(reuslt)
```

* 上面函数中将元组赋值给`(1, 2, 3)`列表，类型会报错。但是函数`avg`实际是可以操作元组的，所以可以改为。

使用接口注解

```python
from collections.abc import Iterable

def avg(items: Iterable[int | float]) -> float:
    total = 0
    count = 0
    for item in items:
        total += item
        count += 1
    return total / count

reuslt = avg((1.5, 2.5, 5.5))
print(reuslt)
```

* `Iterable`为可迭代对象，可以兼容列表、元组和字典等多种数据类型。
* 返回值的范围为`float`，因为均值计算结果，一般就是浮点数。
